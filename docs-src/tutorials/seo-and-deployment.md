# SEO and GitHub Pages Deployment

This tutorial documents the implemented SEO and deployment strategy for this portfolio.

## GitHub Pages base path

The site is published at:

```text
https://danielemasone.github.io/ingdanielemasone/
```

The GitHub Pages project-site base path is:

```text
/ingdanielemasone/
```

`vite.config.mjs`, the router basename and Playwright base URL must preserve this path.

## Static route HTML

`npm run build` runs Vite and then `scripts/prepare-github-pages.mjs`.

The script reads:

- `src/config/seo.json`;
- Italian SEO translations from `src/locales/it/translation.json`;
- the built `dist/index.html` template.

It writes route-specific `index.html` files for configured portfolio routes, then writes `dist/sitemap.xml` and `dist/robots.txt`.

This gives GitHub Pages clean route URLs such as:

```text
https://danielemasone.github.io/ingdanielemasone/github-projects/
```

## Route metadata

Indexable portfolio routes receive:

- localized title and description;
- canonical URL;
- robots directive;
- Open Graph and Twitter metadata;
- social preview image metadata;
- JSON-LD structured data.

`SeoHead` applies route metadata at runtime and removes static fallback tags after React mounts, so the document keeps one canonical metadata set.

## Sitemap and robots

The sitemap contains only routes marked with `sitemap: true` in `src/config/seo.json`.

Legal pages are intentionally excluded from the sitemap and use `noindex, follow`.

`robots.txt` allows the project path and references:

```text
https://danielemasone.github.io/ingdanielemasone/sitemap.xml
```

Generated `/docs/` and `/test-coverage/` resources are not portfolio landing pages and are not included in the public sitemap.

## Developer reports noindex

`scripts/prepare-published-reports.mjs` processes generated `docs/` and `coverage/` HTML files.

It inserts or replaces:

```html
<meta name="robots" content="noindex, nofollow"/>
```

Then it copies:

- `docs/` to `dist/docs/`;
- `coverage/` to `dist/test-coverage/`.

Tutorial pages generated by JSDoc are inside `docs/`, so they receive the same noindex treatment during `npm run build:all`.

## Google Search Console

Use a URL-prefix property:

```text
https://danielemasone.github.io/ingdanielemasone/
```

The HTML verification file lives in `public/` and is published at the property root.

Submit this sitemap URL:

```text
https://danielemasone.github.io/ingdanielemasone/sitemap.xml
```

If a page is not indexed, inspect the generated route HTML in `dist/<route>/index.html` and verify:

- canonical URL includes `/ingdanielemasone/`;
- robots is `index, follow`;
- the route appears in `dist/sitemap.xml`;
- `robots.txt` does not block the project path;
- the deployed URL has a trailing slash and returns the expected route HTML.

## CI deployment flow

`.github/workflows/deploy-pages.yml` runs on pushes to `main` and can be started manually.

The workflow:

1. checks out the repository;
2. configures Node.js;
3. configures GitHub Pages;
4. installs dependencies with `npm ci`;
5. installs the Playwright Chromium headless shell;
6. runs `npm run test:e2e` against production preview;
7. runs `npm run build:reports`;
8. uploads `dist`;
9. deploys through the official GitHub Pages action.

The deployment artifact is only `dist`.

## Local inspection

Useful commands:

```bash
npm run build
npm run build:reports
npm run doc
npm run build:all
```

After `npm run build:all`, inspect:

- `dist/index.html`;
- route-specific `dist/**/index.html`;
- `dist/sitemap.xml`;
- `dist/robots.txt`;
- `dist/docs/index.html`;
- generated tutorial pages in `dist/docs/`;
- `dist/test-coverage/index.html`.
